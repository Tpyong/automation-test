# 系统架构文档

## 1. 系统概述

本项目是一个基于 pytest-playwright 插件的自动化测试框架，用于执行端到端测试、API 测试和集成测试。系统采用模块化设计，具有高度的可扩展性和可维护性，支持多种测试类型和场景。

## 2. 系统架构

### 2.1 整体架构

系统采用分层架构设计，从上到下分为：

1. **测试层**：包含各种类型的测试用例，如 E2E 测试、API 测试、集成测试和单元测试
2. **页面对象层**：包含基础页面类和具体页面类，封装了页面操作和元素定位
3. **服务层**：包含 API 服务、数据库服务等，提供测试所需的各种服务
4. **工具层**：包含浏览器工具、数据管理工具、报告工具等，提供测试所需的各种工具
5. **配置层**：包含配置管理、环境变量管理等，提供系统配置
6. **模型层**：包含数据模型和异常模型，提供数据结构和异常处理

### 2.2 核心模块

#### 2.2.1 测试模块（tests/）

- **tests/e2e/**：端到端测试，使用 pytest-playwright 插件执行浏览器测试
- **tests/api/**：API 测试，测试 API 接口
- **tests/integration/**：集成测试，测试系统各模块的集成
- **tests/unit/**：单元测试，测试单个模块的功能

#### 2.2.2 核心模块（core/）

- **core/pages/**：页面对象，包含基础页面类和具体页面类
  - **core/pages/base/**：基础页面类，提供通用页面操作
  - **core/pages/specific/**：具体页面类，封装特定页面的操作
  - **core/pages/locators.py**：定位器管理，支持多种定位策略（CSS、XPath、语义化定位等）

- **core/services/**：服务模块，包含 API 服务、数据库服务等
  - **core/services/api/**：API 服务，封装 API 调用和 Mock 服务

- **core/models/**：数据模型，包含测试数据模型和业务数据模型

- **core/exceptions/**：异常模块，定义系统异常

#### 2.2.3 工具模块（utils/）

- **utils/browser/**：浏览器工具，包含浏览器池和智能等待器
  - **utils/browser/browser_pool.py**：浏览器池管理，优化浏览器启动和管理
  - **utils/browser/smart_waiter.py**：智能等待器，提供各种等待方法

- **utils/data/**：数据管理工具，包含测试数据加载和管理
  - **utils/data/test_data_loader.py**：测试数据加载器，支持 YAML/JSON/Excel 格式
  - **utils/data/data_factory.py**：测试数据工厂，动态生成测试数据
  - **utils/data/test_data_manager.py**：测试数据管理器，管理测试数据生命周期

- **utils/reporting/**：报告工具，包含 Allure 报告生成和管理
  - **utils/reporting/allure_helper.py**：Allure 报告辅助工具
  - **utils/reporting/report_generator.py**：测试报告生成器

- **utils/api/**：API 工具，包含 API 客户端和合约测试

- **utils/common/**：通用工具，包含日志记录和路径管理

- **utils/security/**：安全工具，包含数据脱敏和密钥管理
  - **utils/security/secrets_manager.py**：密钥管理器，安全管理密钥和凭证

#### 2.2.4 配置模块（config/）

- **config/settings.py**：配置管理，加载和验证配置
- **config/settings_dir/**：环境配置，包含不同环境的配置
  - **config/settings_dir/base/**：基础配置
  - **config/settings_dir/development/**：开发环境配置
  - **config/settings_dir/testing/**：测试环境配置
  - **config/settings_dir/production/**：生产环境配置
- **config/envs/**：环境变量配置，包含不同环境的环境变量
  - **config/envs/.env.base**：基础配置（所有环境的默认值）
  - **config/envs/.env.development**：开发环境配置
  - **config/envs/.env.testing**：测试环境配置
  - **config/envs/.env.production**：生产环境配置
- **config/validators.py**：配置验证器，验证配置的正确性

#### 2.2.5 脚本模块（scripts/）

- **scripts/automation/**：自动化脚本，包含 Allure 报告准备等
- **scripts/cli/**：命令行工具，包含配置向导和交互式运行器
  - **scripts/cli/config_wizard.py**：配置向导，帮助用户配置系统
  - **scripts/cli/interactive_runner.py**：交互式测试运行器
- **scripts/security/**：安全脚本，包含安全扫描等
- **scripts/utils/**：脚本工具，包含配置检查等

#### 2.2.6 资源模块（resources/）

- **resources/data/**：测试数据，包含测试数据集和 fixtures
- **resources/locators/**：定位器配置，包含不同页面的定位器
- **resources/templates/**：模板资源，包含测试模板

## 3. 数据流

### 3.1 测试执行流程

1. **配置加载**：从环境变量和配置文件加载配置，支持配置模式（strict/relaxed）
2. **测试初始化**：初始化测试环境，准备测试数据，启动 Mock 服务（如果需要）
3. **测试执行**：执行测试用例，操作页面或调用 API
4. **结果收集**：收集测试结果和截图、视频等附件
5. **报告生成**：生成 Allure 报告和自定义报告
6. **资源清理**：清理测试数据和资源，停止 Mock 服务

### 3.2 页面对象使用流程

1. **页面初始化**：创建页面对象实例，初始化定位器
2. **页面导航**：导航到目标页面，使用智能等待确保页面加载完成
3. **元素操作**：操作页面元素，如输入、点击等，使用智能定位器提高稳定性
4. **断言验证**：验证页面状态和元素状态
5. **页面退出**：退出页面或关闭浏览器

### 3.3 配置管理流程

1. **配置加载**：按优先级加载配置（环境变量 > 特定环境配置 > .env 文件 > 基础配置 > 默认值）
2. **配置验证**：验证配置的有效性，根据配置模式（strict/relaxed）处理错误
3. **配置使用**：在测试和工具中使用配置
4. **配置变更日志**：记录配置变更历史

## 4. 技术栈

- **测试框架**：pytest + pytest-playwright 插件

## 6. 核心特性

### 6.1 pytest-playwright 集成

- **浏览器自动化**：使用 Playwright 提供的强大浏览器自动化能力
- **多浏览器支持**：支持 Chrome、Firefox、Safari 等主流浏览器
- **并行测试**：支持并行执行测试，提高测试速度
- **智能等待**：自动等待元素可见和可交互，提高测试稳定性

### 6.2 配置管理增强

- **配置模式**：支持 strict/relaxed 两种配置模式
- **配置验证**：自动验证配置的有效性
- **配置变更日志**：记录配置变更历史
- **环境变量管理**：优化环境变量加载和处理，支持敏感信息加密

### 6.3 智能定位器

- **多种定位策略**：支持 CSS、XPath、语义化定位、数据属性定位等
- **定位器管理**：集中管理定位器，提高测试可维护性
- **定位器优先级**：支持定位器优先级，提高定位成功率

### 6.4 浏览器池与智能等待

- **浏览器池**：优化浏览器启动和管理，减少测试执行时间
- **智能等待**：自动等待元素可见和可交互，提高测试稳定性
- **显式等待**：提供灵活的等待方法，满足不同场景需求
- **超时管理**：统一的超时配置，便于维护

### 6.5 Mock 服务

- **模拟 API 响应**：返回自定义响应数据，无需真实 API
- **请求验证**：验证请求参数和 headers
- **延迟响应**：模拟网络延迟，测试系统在不同网络条件下的表现
- **错误响应**：模拟各种错误状态码，测试错误处理逻辑

### 6.6 测试数据管理

- **数据驱动测试**：支持 YAML/JSON/Excel 格式的测试数据
- **测试数据工厂**：动态生成测试数据，减少测试数据维护成本
- **测试数据清理**：自动清理测试数据，确保测试环境干净
- **测试数据隔离**：确保测试之间数据隔离，避免测试相互影响

### 6.7 增强的报告系统

- **Allure 报告**：详细的测试报告和趋势分析
- **自定义报告**：HTML 和 JSON 格式的测试摘要
- **测试趋势**：历史测试结果分析，识别测试稳定性问题
- **智能测试建议**：基于历史数据的测试优化建议

### 6.8 安全管理

- **敏感信息加密**：加密环境变量中的敏感信息
- **审计日志**：记录敏感操作，便于安全审计
- **数据掩码**：在报告中掩码敏感数据，保护隐私
- **合规检查**：检查代码中的安全问题，确保测试代码安全

本系统采用模块化、分层架构设计，具有高度的可扩展性和可维护性。通过使用 pytest-playwright 插件，实现了高效、稳定的自动化测试。系统支持多种测试类型，包括 E2E 测试、API 测试、集成测试和单元测试，能够满足不同场景的测试需求。

系统的核心特性包括：
- 完整的 pytest-playwright 集成
- 增强的配置管理系统
- 智能定位器和浏览器池
- 内置 Mock 服务
- 完善的测试数据管理
- 增强的报告系统
- 安全管理和监控

这些特性使得系统能够适应复杂的测试场景，提供可靠的测试结果，为项目质量保驾护航。
