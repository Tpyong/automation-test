# 故障排查手册

## 1. 概述

本文档描述了系统常见故障的排查方法和解决方案，旨在帮助开发人员和运维人员快速定位和解决问题。

## 2. 常见故障及解决方案

### 2.1 浏览器相关故障

#### 2.1.1 浏览器启动失败

**症状**：测试执行时浏览器无法启动，出现错误信息。

**可能原因**：
- 浏览器依赖缺失
- 端口被占用
- 权限不足
- 浏览器版本不兼容

**解决方案**：
1. **重新安装 Playwright 浏览器**：
   ```bash
   python -m playwright install --with-deps
   ```

2. **检查端口占用情况**：
   ```bash
   # Linux/macOS
   lsof -i :9222
   
   # Windows
   netstat -ano | findstr :9222
   ```

3. **以管理员权限运行测试**：
   ```bash
   # Linux/macOS
   sudo pytest
   
   # Windows
   # 以管理员身份运行命令提示符
   pytest
   ```

4. **检查浏览器版本**：
   ```bash
   python -m playwright --version
   ```

#### 2.1.2 浏览器崩溃

**症状**：测试执行过程中浏览器崩溃，出现错误信息。

**可能原因**：
- 内存不足
- 浏览器进程被终止
- 测试代码存在问题

**解决方案**：
1. **增加内存**：确保系统有足够的内存
2. **检查测试代码**：检查测试代码是否存在问题
3. **减少并行测试数量**：
   ```bash
   pytest -n 2  # 减少并行进程数
   ```

#### 2.1.3 浏览器超时

**症状**：浏览器操作超时，出现错误信息。

**可能原因**：
- 网络连接慢
- 页面加载时间长
- 元素定位失败

**解决方案**：
1. **增加超时时间**：修改 `TIMEOUT` 环境变量
2. **检查网络连接**：确保网络连接正常
3. **优化元素定位策略**：使用更稳定的定位策略
4. **使用智能等待**：使用 `SmartWaiter` 进行智能等待

### 2.2 测试相关故障

#### 2.2.1 测试失败

**症状**：测试执行失败，出现错误信息。

**可能原因**：
- 测试代码存在问题
- 测试数据不正确
- 测试环境配置错误
- 被测试系统有问题

**解决方案**：
1. **查看测试日志**：查看详细的测试日志，了解失败原因
2. **检查测试代码**：检查测试代码是否存在问题
3. **检查测试数据**：确保测试数据正确
4. **检查测试环境**：确保测试环境配置正确
5. **检查被测试系统**：确保被测试系统正常运行

#### 2.2.2 测试超时

**症状**：测试执行超时，出现错误信息。

**可能原因**：
- 网络连接慢
- 页面加载时间长
- 元素定位失败
- 测试逻辑复杂

**解决方案**：
1. **增加超时时间**：修改 `TIMEOUT` 环境变量
2. **优化测试逻辑**：简化测试逻辑，减少测试步骤
3. **使用并行测试**：使用 `-n` 参数进行并行测试
4. **使用标记**：使用标记运行特定测试

#### 2.2.3 测试不稳定

**症状**：测试有时通过，有时失败，结果不稳定。

**可能原因**：
- 网络波动
- 页面加载时间不稳定
- 元素定位不稳定
- 测试数据不稳定

**解决方案**：
1. **使用智能等待**：使用 `SmartWaiter` 进行智能等待
2. **使用稳定的定位策略**：使用语义化定位
3. **固定测试数据**：使用固定的测试数据
4. **增加重试机制**：使用 `pytest-rerunfailures` 插件

### 2.3 配置相关故障

#### 2.3.1 配置加载失败

**症状**：配置加载失败，出现错误信息。

**可能原因**：
- 环境变量配置错误
- 配置文件格式错误
- 配置项缺失

**解决方案**：
1. **检查环境变量**：确保环境变量配置正确
2. **检查配置文件**：确保配置文件格式正确
3. **检查配置项**：确保所有必要的配置项都存在
4. **使用配置检查器**：运行配置检查器检查配置

#### 2.3.2 配置验证失败

**症状**：配置验证失败，出现错误信息。

**可能原因**：
- 配置值无效
- 配置值超出范围
- 配置值格式错误

**解决方案**：
1. **检查配置值**：确保配置值有效
2. **检查配置范围**：确保配置值在有效范围内
3. **检查配置格式**：确保配置值格式正确
4. **使用配置检查器**：运行配置检查器检查配置

### 2.4 报告相关故障

#### 2.4.1 报告生成失败

**症状**：Allure 报告生成失败，出现错误信息。

**可能原因**：
- Allure 命令行工具未安装
- 报告目录权限不足
- 测试结果文件损坏
- 测试结果目录不存在

**解决方案**：
1. **安装 Allure 命令行工具**：
   ```bash
   # Linux/macOS
   wget -qO- https://github.com/allure-framework/allure2/releases/download/2.24.1/allure-2.24.1.tgz | tar -xz -C /tmp/
   export PATH=$PATH:/tmp/allure-2.24.1/bin
   
   # Windows
   # 下载 Allure 命令行工具并添加到 PATH
   ```

2. **检查目录权限**：确保报告目录有写权限
3. **清理测试结果目录**：
   ```bash
   rm -rf allure-results
   ```

4. **确保测试结果目录存在**：
   ```bash
   mkdir -p allure-results
   ```

#### 2.4.2 报告内容不完整

**症状**：Allure 报告生成成功，但内容不完整。

**可能原因**：
- 测试结果文件不完整
- 测试执行中断
- 报告生成过程中断

**解决方案**：
1. **重新运行测试**：确保测试执行完整
2. **检查测试结果文件**：确保测试结果文件完整
3. **重新生成报告**：
   ```bash
   allure generate allure-results --clean -o allure-report
   ```

### 2.5 依赖相关故障

#### 2.5.1 依赖安装失败

**症状**：依赖安装失败，出现错误信息。

**可能原因**：
- 网络连接问题
- 依赖包版本冲突
- 系统依赖缺失

**解决方案**：
1. **检查网络连接**：确保网络连接正常
2. **清理 pip 缓存**：
   ```bash
   pip cache purge
   ```

3. **使用指定版本**：在 `requirements.in` 中指定依赖版本
4. **安装系统依赖**：
   ```bash
   # Linux
   sudo apt-get update && sudo apt-get install -y <system-dependencies>
   
   # macOS
   brew install <system-dependencies>
   ```

#### 2.5.2 依赖版本冲突

**症状**：依赖版本冲突，出现错误信息。

**可能原因**：
- 多个依赖包要求不同版本的同一个包
- 依赖包版本不兼容

**解决方案**：
1. **使用 pip-compile**：
   ```bash
   pip-compile --upgrade requirements.in
   ```

2. **指定兼容版本**：在 `requirements.in` 中指定兼容的版本
3. **使用虚拟环境**：使用虚拟环境隔离依赖

### 2.6 安全相关故障

#### 2.6.1 安全扫描失败

**症状**：安全扫描失败，出现错误信息。

**可能原因**：
- 依赖包存在安全漏洞
- 代码存在安全问题
- 安全扫描工具配置错误

**解决方案**：
1. **更新依赖包**：更新存在安全漏洞的依赖包
2. **修复代码安全问题**：修复代码中存在的安全问题
3. **检查安全扫描工具配置**：确保安全扫描工具配置正确

#### 2.6.2 敏感信息泄露

**症状**：敏感信息泄露，出现在日志或报告中。

**可能原因**：
- 硬编码敏感信息
- 日志记录敏感信息
- 报告中包含敏感信息

**解决方案**：
1. **使用环境变量**：使用环境变量存储敏感信息
2. **数据脱敏**：对敏感数据进行脱敏处理
3. **配置日志级别**：避免在日志中记录敏感信息
4. **清理报告**：确保报告中不包含敏感信息

## 3. 故障排查工具

### 3.1 日志分析工具

- **pytest 日志**：查看 pytest 执行日志
  ```bash
  pytest -v --tb=long
  ```

- **浏览器日志**：查看浏览器控制台日志
  ```python
  # 在测试代码中添加
  console_logs = page.context.logs()
  for log in console_logs:
      print(log)
  ```

- **系统日志**：查看系统日志
  ```bash
  # Linux
  tail -f /var/log/syslog
  
  # macOS
  tail -f /var/log/system.log
  
  # Windows
  eventvwr.msc
  ```

### 3.2 调试工具

- **交互式调试**：使用 pytest 的交互式调试
  ```bash
  pytest -v -s
  ```

- **断点调试**：使用 IDE 的断点调试功能

- **浏览器开发者工具**：使用浏览器的开发者工具

### 3.3 性能分析工具

- **cProfile**：Python 性能分析工具
  ```bash
  python -m cProfile -o profile.out pytest tests/e2e/test_todomvc_example.py
  ```

- **memory_profiler**：内存分析工具
  ```bash
  pip install memory_profiler
  python -m memory_profiler pytest tests/e2e/test_todomvc_example.py
  ```

### 3.4 安全扫描工具

- **safety**：依赖包安全扫描
  ```bash
  safety check
  ```

- **bandit**：代码安全扫描
  ```bash
  bandit -r core/ tests/
  ```

- **pip-audit**：依赖包审计
  ```bash
  pip-audit
  ```
